# SPEEDKIT Control Engine

SPEEDKIT is the public control engine for the SPEEDKIT registry surface.

It has two separate layers.

## Recognized execution systems

These are admitted systems with execution identity, boundary, registry standing, verification/replay path, and public control surface.

Current recognized execution systems:

- CICULLIS
- MK10-PRO
- VERIFRAX
- VATFIX

Current count: 4.

## Public workspace universe

These are public surfaces exposed for routing, search, inspection, verification, and export.

They include:

- GitHub users and organizations
- GitHub repositories
- npm packages
- PyPI packages
- workspace family pages
- route pages
- source and distribution surfaces

## Boundary rule

A repository is not automatically a system.

An organization is not automatically a system.

A package is not automatically a system.

Public existence is metadata.

System standing requires admission.

## Surface language rule

The SPEEDKIT surface must use canonical registry language only.

The canonical language is:

- SPEEDKIT Control Engine
- Public Control Engine
- Public Workspace Universe
- Recognized Execution Systems
- Public-Only Gate
- Searchable, inspectable, verifiable, exportable control surface

Subjective promotional status language is not valid control-surface language.
