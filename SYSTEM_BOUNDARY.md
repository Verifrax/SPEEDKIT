# SPEEDKIT System Boundary

SPEEDKIT has two different layers.

## Recognized execution systems

These are formally admitted systems with registry standing:

- CICULLIS
- MK10-PRO
- VERIFRAX
- VATFIX

Current count: 4.

## Public workspace universe

These are public surfaces, but they are not automatically systems:

- GitHub users
- GitHub organizations
- GitHub repositories
- npm packages
- PyPI packages
- workspace families
- route pages
- source/distribution surfaces

Current public surface count: 135.

## Rule

A repository is not automatically a system.

An organization is not automatically a system.

A package is not automatically a system.

A system requires execution identity, boundary, registry standing, verification/replay path, and public control surface.
