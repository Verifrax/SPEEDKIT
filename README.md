# SPEEDKIT

```
SYS-000
STATUS: REGISTERED
REGISTRY: https://speedkit.eu
```

## Purpose

Public registry of recognized systems in the Verifrax stack.

SPEEDKIT is the public recognition and routing boundary for systems that have been admitted into registry status.
It is not constitutional law.
It is not accepted state.
It is not reconciliation runtime.
It is not verifier authority.

## Stack position

- SYNTAGMARIUM legislates
- ORBISTIUM stores accepted state
- CONSONORIUM reconciles
- TACHYRIUM computes
- VERIFRAX verifies
- SPEEDKIT recognizes and routes publicly admitted systems

## Authority

SPEEDKIT is authoritative only for registry recognition metadata published by this repository.

Authoritative registry fields:

- canonical system name
- registry class
- registry status
- discovery URL
- entry point URL
- authoritative specification URL

## Registry classes

- recognized
- superseded
- discontinued

## Policy

- append-only
- no deletion
- public read-only
- recognition only
- routing only

## Non-authority

SPEEDKIT does not define law.
SPEEDKIT does not determine verification truth.
SPEEDKIT does not store accepted world-state.
SPEEDKIT does not execute reconciliation.
SPEEDKIT does not own repository identities outside this repository.

## Admission rule

A system may be admitted only if it has:

- canonical name
- bounded purpose
- discovery URL
- entry point URL
- authoritative specification URL
- execution boundary
- output or proof surface
- termination condition
- access rule
- archive guarantee
- registry class
- registry status
